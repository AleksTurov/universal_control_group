# Universal Control Group for Churn & CVM

## О проекте

Проект создает универсальную контрольную группу (УКГ) для корректной оценки инкрементального эффекта CVM-активностей по всей базе абонентов.

УКГ формируется как единая глобальная `holdout`-группа:

- активные абоненты
- неактивные абоненты

Абоненты из УКГ исключаются из всех целевых коммуникаций. При этом для анализа конкретной кампании используется не вся УКГ целиком, а только ее релевантный срез, соответствующий правилам отбора данной кампании.

Иными словами:

- глобальная УКГ нужна для операционного исключения из коммуникаций
- для каждой кампании строится свой `control slice` внутри этой УКГ

Это позволяет одновременно:

- иметь единый стабильный baseline для всей CVM-системы
- корректно измерять эффект отдельных кампаний
- накапливать историю `treatment/control` для последующего uplift modeling

## Цели

- Получать корректную оценку эффекта churn-кампаний без смещения.
- Создать единый глобальный holdout для всех CVM-коммуникаций.
- Стабильно поддерживать контрольную группу при изменении базы.
- Подготовить чистую экспериментальную инфраструктуру для последующего uplift modeling в отдельном треке.

## Scope

- Построение и поддержка УКГ на всей eligible-базе: активные и неактивные абоненты после исключений.
- Ежемесячный добор только новых абонентов.
- Глобальное исключение УКГ из всех CVM-коммуникаций.
- Использование релевантного `control slice` из УКГ для оценки каждой отдельной кампании.
- Контроль баланса по ключевым сегментам.

## Out of Scope

- Uplift modeling.
- Оптимизация коммуникационной политики по uplift-скорингу.
- Пересборка исторической УКГ задним числом.

## Принцип работы

1. На всей eligible-базе формируется единая глобальная УКГ.
2. Абоненты из УКГ исключаются из любых целевых CVM-коммуникаций.
3. Для каждой кампании определяется ее целевая популяция (`eligible population`).
4. Для оценки эффекта кампании выбирается соответствующий `control slice` из УКГ по тем же правилам отбора.
5. Сравнение выполняется внутри сопоставимой популяции, а не со всей УКГ целиком.

Пример:

- для `churn`-кампании по активным абонентам контроль берется из активной части УКГ, подходящей под правила кампании
- для `reactivation`-кампании контроль берется из неактивной части УКГ, подходящей под правила кампании

## Дизайн УКГ

- Размер УКГ: `5-10%` от всей eligible-базы.
- Единица назначения: `subscriber_id`.
- Рандомизация: детерминированная через `hash(subscriber_id + salt)`.
- Назначение постоянное: абонент не перекидывается между группами.
- Пополнение ежемесячное: добавляются только новые абоненты.
- Исторические назначения не пересчитываются.
- База отбора: вся eligible-база, включая активных и неактивных абонентов, после исключений.

## Почему одна глобальная УКГ

Подход с одной глобальной УКГ нужен для того, чтобы:

- избежать пересечений и конфликтов между контрольными группами разных кампаний
- не создавать отдельную контрольную группу под каждую коммуникацию
- иметь единое операционное правило исключения из CVM
- измерять эффект разных кампаний на общей экспериментальной основе

Важно:

- одна глобальная УКГ не означает, что любую кампанию нужно сравнивать со всей УКГ
- корректное сравнение всегда делается только с релевантным `control slice`

## Исключения

- Сотрудники.
- Тестовые номера.
- `M2M`.
- Иные некоммуникабельные категории по согласованному списку.

## Артефакты в репозитории

На текущем этапе репозиторий содержит описание методологии и целевого процесса построения УКГ.

SQL-реализация находится в статусе подготовки и будет добавлена отдельно:

- DDL таблиц УКГ
- скрипт ежемесячного обновления УКГ
- скрипт проверок качества распределения

## Процесс запуска

Текущий статус: логика и требования описаны, SQL-реализация еще не добавлена.

### 1. Инициализация

Будет выполняться один раз после подготовки DDL и таблиц.

### 2. Ежемесячное обновление

Будет выполняться после подготовки SQL-пайплайна обновления УКГ.

Параметры:

- `REPORT_DT`: дата среза.
- `UKG_PCT`: доля УКГ, например `0.07`.
- `UKG_SALT`: фиксированная соль назначения.
- `ASSIGNMENT_VERSION`: версия логики назначения.

### 3. Контроль качества

Будет выполняться после подготовки SQL-проверок качества.

Проверки:

- Фактическая доля УКГ по всей eligible-базе.
- Баланс активных и неактивных абонентов.
- Баланс по `contract_status`.
- Баланс по крупным тарифным сегментам.
- Отсутствие дублей в `current`-таблице.
- Корректность выделения `control slice` для аналитики кампаний.

## Правила аналитического использования

- Глобальная УКГ используется как единое исключение из коммуникаций.
- Для оценки эффекта конкретной кампании используется только релевантный `control slice`.
- Сравнение с полной УКГ без учета правил отбора кампании не допускается.
- Анализ эффекта должен проводиться минимум по тем срезам, которые участвуют в логике таргетинга кампании.
- Для aggregate-отчетности результаты могут быть дополнительно агрегированы на уровень всей базы, но только после расчета внутри корректных популяций.

## Эксплуатационные правила

- Не менять `UKG_SALT` без миграционного плана.
- Не переназначать исторических абонентов.
- Любое изменение логики фиксировать через `ASSIGNMENT_VERSION`.
- Перед запуском кампаний использовать `current`-таблицу для глобального исключения УКГ.
- Не использовать всю УКГ как прямой control для кампаний с узкой целевой популяцией.

## Связь с churn-инициативой

- Текущий трек: churn-модель и измерение эффекта кампаний.
- Следующий трек: uplift по churn после накопления стабильной истории `treatment/control`.

Глобальная УКГ позволяет на одной и той же инфраструктуре измерять, например:

- `churn prevention`
- `reactivation`
- другие CVM-кампании

Но каждая из них должна анализироваться на своем релевантном срезе контрольной группы.

## Роли и ответственность

- `Marketing/CVM`: правила кампаний, исключение УКГ из коммуникаций, бизнес-KPI.
- `DS/Analytics`: методология эксперимента, SRM-контроль, формирование `control slice`, эффект и отчетность.
- `DWH/DE`: стабильная витрина базы, ежемесячный pipeline, контроль качества данных.

## Критерии готовности

- УКГ создана и стабильно пополняется ежемесячно.
- Доля УКГ соответствует целевому диапазону `5-10%`.
- Баланс по ключевым срезам находится в допустимых отклонениях.
- УКГ глобально исключается из CVM-активностей.
- Для каждой кампании используется корректный `control slice`, а не вся УКГ.
- Есть регулярный отчет по качеству групп и эффекту кампаний.
